## Workflow

1. Make changes to the code based on your task. Any changes in this repository 
should happen inside the `src/` directory. 
2. Changes in `src/` require the shortcut to be rebuilt using `src/build.py`. Re
build the shortcut and make sure there are no errors.
3. Write tests to validate the behavior of builds and the shortcut itself.
4. Whenever you make changes to the code, make sure those changes are accurately 
reflected in the `README.md` documentation.
